# 01. 제품 기획서 (PRD)

## 1. 목적

영어 초급 학습자(한국어 모국어)가 **문장성분**(주어·서술어·목적어·보어·수식어 등)과 **품사**(명사·대명사·동사·형용사·부사·전치사·접속사)를 스스로 풀고 즉시 피드백받으며 익히는 웹 앱.

기존 종이 워크북(`resource_docs/워크북 문제_*.pdf`)의 학습 흐름을 그대로 디지털화하되, 채점·재시도·색상 코딩·한글 해석·완료 보상을 자동화한다.

## 2. 대상 사용자

- 1차 사용자: 개인(앱 소유자) 및 그 자녀/학생. **개인용**이므로 회원가입·서버·다중 사용자 동기화는 불필요.
- 기기: 데스크톱 우선, 모바일/태블릿 대응(반응형). `design.md`의 Breakpoints 준수.

## 3. 핵심 퀴즈 진행 흐름 (4단계)

사용자가 요구한 진행 순서를 그대로 따른다.

### 단계 1 — 문제 제시 & 입력
- 종이 워크북과 동일한 형태로 한 문장을 제시한다.
  - 문장은 단어 단위로 쪼개지고, 각 단어 위에 **번호(index)** 가 붙는다.
  - 각 단어마다 **품사 선택기**, 각 구(phrase)마다 **문장성분 선택기**를 제공한다.
- 이 단계의 보드는 **무채색**(품사 틴트/문장성분 색 없음). 색은 정답 공개 시에만 등장.
- 모든 칸을 채우기 전까지 "채점하기" 버튼은 비활성.

### 단계 2 — 채점 & 오답 표시
- "채점하기"를 누르면 채점.
- 틀린 부분만 **빨갛게**(`error`/`error-soft`) 표시하고, "N곳이 틀렸어요. 빨간 부분을 다시 생각해 볼까요?" 식으로 재시도를 유도한다.
- **정답은 아직 공개하지 않는다.** 어디가 틀렸는지(위치)만 알려준다.
- 학습자가 답을 고쳐 다시 채점할 수 있다(반복 가능).

### 단계 3 — 전부 정답 시 공개(Reveal)
- 모든 답이 맞으면:
  - 전체를 정답으로 표시하되, **원래 틀렸던 칸**에는 작은 조용한 체크 마커(`was-wrong`)를 남겨 "고쳐서 맞힌 부분"을 알 수 있게 한다.
  - 문장에 **색상 코딩** 적용: 단어마다 품사 틴트, 구마다 문장성분 밑줄 브래킷 + 라벨 칩.
  - 각 구(문장성분 단위) 아래에 **한글 해석(chunk)** 을 단다.
  - 그 아래에 문장 전체를 **한글 문법 순서로 재배열한 해석(문장)** 을 크림색 콜아웃에 단다.

### 단계 4 — 완료 & 보상
- 정해진 문제 수를 모두 풀면 최종 점수 화면.
- **정답률(틀린 적 없이 한 번에 맞힌 비율 기준 권장)** 에 따라 보상 차등:
  - 상위(예: ≥90%): 풀스크린 코랄 카드 + 폭죽(fireworks).
  - 중위(예: ≥70%): 포레스트 카드 + 트로피 드롭.
  - 하위: 다크 카드 + 응원 캐릭터/인형.
- 보상 연출은 결과 카드 안에서만, 짧고 비차단(non-blocking)으로. (`design.md` Motion & Reward)

> 점수 산정 기준은 [02_데이터모델.md](02_데이터모델.md) "채점·점수 규칙" 참고.

## 4. 화면 목록

| 화면 | 설명 | 주요 컴포넌트(design.md) |
|---|---|---|
| 시작 화면 | 타이틀 + 스토리/덱 선택 + "시작하기" | `start-hero`, `deck-card` |
| 퀴즈 보드 | 문장 + 선택기 + 채점/다음 | `quiz-board`, `word-token`, `pos-selector`, `component-selector`, `progress-indicator` |
| 채점 피드백 | 오답 빨강 표시 + 재시도 안내 | `grade-summary-inline` |
| 정답 공개 | 색상 코딩 + chunk 해석 + 문장 해석 | `reveal-panel`, `component-bracket`, `chunk-gloss`, `translation-callout`, `legend` |
| 결과/축하 | 점수 + 보상 연출 | `result-card`, `reward-animation` |

## 5. 기능 요구사항

### 필수 (MVP)
- [ ] 저장된 JSON 데이터로 퀴즈 진행(시드 스토리 1편 내장: 생쥐 이야기).
- [ ] 단어별 품사 선택, 구별 문장성분 선택 입력.
- [ ] 채점 → 오답만 빨강 표시 → 재시도 → 전부 정답 시 공개.
- [ ] 공개 시 품사 틴트 + 문장성분 브래킷 + chunk 해석 + 문장 해석 + 원오답 마커.
- [ ] 진행률 표시, 다음 문제 이동.
- [ ] 완료 시 정답률 기반 차등 축하 연출.
- [ ] 품사/문장성분 색상 범례(legend).

### 선택 (Nice to have)
- [ ] 학습 기록(점수 히스토리) 로컬 저장.
- [ ] 문장 발음 오디오 재생(`button-icon-circular`).
- [ ] 스토리 덱 여러 개 관리.

## 6. 비기능 요구사항
- 백엔드 없는 정적 웹앱(개인용).
- 접근성: 키보드 포커스 링(`focus`), 터치 타깃 최소 44~48px.
- 라이트 모드 전용(다크 모드는 범위 외 — design.md Known Gaps).

## 7. 범위 외
- 사용자 계정/서버 동기화/결제.
- 다크 모드.
- 워크북 자동 OCR(문제 데이터는 JSON으로 공급).
