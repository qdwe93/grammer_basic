# 04. 디자인 적용 가이드

`design.md`가 디자인 시스템의 **단일 진실 소스(SSOT)** 다. 이 문서는 그 토큰을 코드(CSS 변수)로 옮기고, 구현 시 자주 틀리는 규칙을 요약한다. 색상/타이포/간격 값은 모두 `design.md`에서 가져왔다.

## 1. 절대 규칙 (먼저 외울 것)

1. **2채널 분리**: 품사(품사) = 단어 **배경 틴트** / 문장성분(문장성분) = 구 아래 **2px 밑줄 브래킷 + 라벨 칩**. 한 채널의 색을 다른 채널에 절대 재사용하지 않는다.
2. **색은 보상**: 답하는 동안 보드는 무채색. 전부 정답 → 색·해석 공개(틴트 fade-in ~200ms).
3. **피드백 색은 채점 전용**: `error`(빨강)=오답, `success`(초록)=정답. 브랜드/문법 색으로 쓰지 않는다.
4. **Primary는 거의 검정**(`#181d26`), 링크 파랑(`#1b61c9`)이 아니다. Primary CTA는 뷰포트당 1개.
5. **그라데이션/메시/오로라 금지**. 표면은 흰색/연회색뿐.
6. **디스플레이 굵게 금지**(400/500). 700 사용 금지. 강조는 크기·색·문법 팔레트로.
7. 보상 모션은 결과 카드 안에서만, 비차단.

```css

  --surface-strong:#e0e2e6;

  --hairline:      #dddddd;

  /* Text */

  --body:  #333840;
```

## 3. 타이포그래피 토큰

| 토큰 | size/weight/line-height | 용도 |
|---|---|---|
| display-xl | 48 / 500 / 1.1 | 결과 점수 숫자 |
| display-lg | 40 / 400 / 1.2 | 시작·화면 타이틀 |
| display-md | 32 / 400 / 1.2 | 결과 카드 헤드라인, 섹션 헤드 |
| sentence | 28 / 400 / 1.5 | **영어 퀴즈 문장**(브래킷 자리 위해 1.5) |
| title-lg | 24 / 400 / 1.35 | 서브섹션·설정 그룹 |
| title-md | 20 / 400 / 1.5 | 카드·모달 타이틀 |
| label-md | 16 / 500 / 1.4 | 선택기·범례 라벨 |
| button | 16 / 500 / 1.4 | CTA 라벨 |
| body-md | 14 / 400 / 1.25 | 안내 문구·설정 도움말 |
| gloss | 14 / 500 / 1.4 | **해석(chunk)** — 브래킷 아래 한글 |
| translation | 16 / 400 / 1.6 | **해석(문장)** — 크림 콜아웃 |
| index | 12 / 500 / 1.2 | 단어 위 번호(muted) |
| chip | 12 / 600 / 1.2 | 문장성분 칩·품사 태그 |

- 폰트: 라틴 = Inter Display(Haas 대체), 한글 = **Pretendard** → fallback `"Apple SD Gothic Neo","Malgun Gothic","Noto Sans KR"`. 한글 line-height는 라틴보다 ~5% 느슨하게.

## 5. 반응형 (design.md Breakpoints 요약)
- **<768 Mobile**: 단일 컬럼, 헤더 햄버거(풀스크린 시트), 선택기 → `selector-tray`, 범례 토글, 결과 카드 풀블리드 유지.
- **768–1024 Tablet**: 인라인 토큰 유지, 선택기 인라인/트레이, 덱 2열.
- **1024–1440 Desktop**: 단어 아래 인라인 선택기, 범례 펼침, 덱 3열, 보드 840px.
- **>1440 Wide**: Desktop과 동일 + 바깥 여백만 증가(읽기 컬럼은 840px 유지).
- 줄바꿈 시 브래킷은 두 세그먼트로 정렬 렌더, chunk gloss는 브래킷에 붙어 따라감.

## 6. 자주 하는 실수 체크리스트
- [ ] pos 틴트를 comp 브래킷에, 또는 그 반대로 쓰지 않았는가.
- [ ] 답하는 중 보드가 무채색인가(색은 reveal에서만).
- [ ] error/success를 브랜드 색으로 쓰지 않았는가.
- [ ] Primary가 검정인가(링크 파랑 아님).
- [ ] 디스플레이 텍스트에 700을 쓰지 않았는가.
- [ ] 그라데이션을 넣지 않았는가.
- [ ] 모든 pos 잉크-온-틴트, comp 칩이 WCAG AA를 통과하는가.
