# 03. 기술 스택 & 아키텍처

> 본 문서의 스택은 **권장안**이며, `design.md`가 웹(CSS 변수·반응형 breakpoint) 기준으로 작성된 점과 "개인용·백엔드 불필요" 요구를 반영해 정했다. 팀/취향에 따라 조정 가능하나, 조정 시 `04_디자인적용가이드.md`의 토큰 매핑은 유지한다.

## 1. 권장 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 빌드/런타임 | **Vite + React 18 + TypeScript** | 정적 SPA, 빠른 개발, 타입 안전(데이터 스키마 검증에 유리) |
| 스타일 | **CSS Variables(디자인 토큰) + 일반 CSS Modules** 또는 Tailwind | `design.md`가 토큰 기반 → CSS 변수와 궁합. Tailwind 사용 시 토큰을 theme에 매핑 |
| 상태 관리 | React 내장(useReducer/Context) 또는 **Zustand** | 퀴즈 진행 상태가 단순·국소적 |
| 데이터 검증 | **Zod** | 02 문서의 스키마 검증 |
| 저장소 | **localStorage** | 학습 기록(개인용) |
| 애니메이션 | CSS transition + 가벼운 라이브러리(예: canvas-confetti) | 보상 연출 |
| 폰트 | Pretendard(한글) + Inter Display(라틴, Haas 대체) | design.md Typography |

## 3. 핵심 상태 모델 (퀴즈)

```ts
type AnswerState = {
  // 학습자 입력
  posByIndex: Record<number, Pos | null>;        // 토큰 index → 선택 품사
  componentByPhrase: Record<string, Component | null>; // phrase id → 선택 문장성분
  // 채점 결과
  graded: boolean;
  wrongTokenIndexes: Set<number>;
  wrongPhraseIds: Set<string>;
  // 이력 (was-wrong 마커 & 점수)
  everWrongTokenIndexes: Set<number>;
  everWrongPhraseIds: Set<string>;
  revealed: boolean;        // 전부 정답 → 공개
};
```

진행 단계(상태 머신): `answering → graded(오답 있음) → answering(재시도) → … → revealed(전부 정답) → next/complete`.
